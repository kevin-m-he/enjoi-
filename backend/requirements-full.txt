# Optional quality upgrades — GPU MusicGen + local Whisper transcription.
# Each maps to a capability flag in /api/health; the app falls back gracefully
# without them (see docs/API_CONTRACT.md rule 2).
#
# ⚠️  Do NOT `pip install -r` this file directly. audiocraft 1.3.0 hard-pins
#     torch==2.1.0 (it would replace a CUDA build with the CPU one) and av==11.0.0
#     (no Windows wheel → tries to compile against FFmpeg and fails). Use the
#     automated installer instead, which performs the steps below in order:
#
#         powershell -ExecutionPolicy Bypass -File scripts\install-ml.ps1
#
# Reproducible recipe (Windows, Python 3.11, verified on an RTX 4080 / cu121):
#
# 1) CUDA PyTorch matched to audiocraft's pin:
#      pip install torch==2.1.0 torchaudio==2.1.0 \
#        --index-url https://download.pytorch.org/whl/cu121
#
# 2) audiocraft's dependency stack, with a modern prebuilt `av` wheel and the
#    2024-era versions it expects (torch held by a constraints file):
#      av  transformers==4.40.2  huggingface_hub==0.23.4  tokenizers==0.19.1
#      xformers==0.0.22.post7  encodec==0.1.1  einops  julius  num2words
#      sentencepiece  omegaconf==2.3.0  hydra-core==1.3.2  hydra_colorlog
#      demucs==4.0.1  torchmetrics  "spacy>=3.6.1,<3.9"  flashy
#      (constraints.txt: torch==2.1.0+cu121 / torchaudio==2.1.0+cu121)
#
# 3) audiocraft itself WITHOUT its deps (so it can't undo step 1/2):
#      pip install audiocraft --no-deps
#
# 4) Local Whisper:
#      pip install faster-whisper
#
# Optional (not required; no reliable Windows wheels / extra setup):
#   torchcrepe>=0.0.22        # higher-accuracy f0 (else librosa.pyin)
#   pyrubberband>=0.4.0       # needs the `rubberband` CLI on PATH
#   madmom / essentia         # analysis upgrades (librosa is the baseline)
